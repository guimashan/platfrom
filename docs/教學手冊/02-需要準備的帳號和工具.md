# 02 - 需要準備的帳號和工具

> 就像組樂高前要先準備好所有零件，開發系統前也要先註冊好所有必要的帳號。

## 📝 需要註冊的帳號清單

### 1. LINE Developers 帳號 ⭐⭐⭐

**用途**：建立 LINE Login、LIFF、Messaging API
**費用**：完全免費
**註冊網址**：https://developers.line.biz/

**註冊步驟：**
1. 用您的 LINE 帳號登入
2. 同意開發者條款
3. 填寫基本資料（姓名、Email）
4. 完成！

**注意事項：**
- ⚠️ 建議使用廟方正式的 LINE 帳號註冊
- ⚠️ 這個帳號會成為系統的擁有者
- ⚠️ 請妥善保管帳號密碼

### 2. Firebase 帳號 ⭐⭐⭐

**用途**：建立後端專案、資料庫、Cloud Functions
**費用**：Blaze 方案（依用量計費，小型專案約 $0-5/月）
**註冊網址**：https://console.firebase.google.com/

**註冊步驟：**
1. 用 Google 帳號登入
2. 第一次使用會要求同意條款
3. 建立專案時需要綁定信用卡（升級到 Blaze 方案）
4. 完成！

**注意事項：**
- ⚠️ 需要有 Google 帳號（Gmail）
- ⚠️ 需要信用卡（但費用很低，可設定預算上限）
- ⚠️ 建議使用廟方的 Google 帳號

### 3. GitHub 帳號 ⭐⭐⭐

**用途**：儲存程式碼、版本控制
**費用**：完全免費
**註冊網址**：https://github.com/

**註冊步驟：**
1. 填寫 Email、使用者名稱、密碼
2. 驗證 Email
3. 完成！

**注意事項：**
- ⚠️ 使用者名稱會出現在網址裡，建議用廟名（例如：guimashan）
- ⚠️ 不要忘記密碼！建議寫在安全的地方

### 4. Vercel 帳號 ⭐⭐⭐

**用途**：部署前端網站到正式環境
**費用**：Pro 方案 $20/月（已升級，無部署次數限制）
**註冊網址**：https://vercel.com/

**註冊步驟：**
1. 用 GitHub 帳號登入（建議）或用 Email 註冊
2. 選擇個人帳號或團隊帳號
3. 完成！

**注意事項：**
- ⚠️ 建議用 GitHub 帳號登入，比較方便
- ⚠️ Pro 方案需要信用卡

### 5. Replit 帳號（選用）⭐⭐

**用途**：線上開發環境（目前使用中）
**費用**：基本功能免費
**註冊網址**：https://replit.com/

**註冊步驟：**
1. 用 Google 或 GitHub 帳號登入
2. 完成！

**注意事項：**
- ⚠️ 如果您已經在 Replit 上看到這份文件，就不需要再註冊了！

## 🛠️ 需要安裝的軟體（本機開發用）

### 如果您使用 Replit（推薦新手）
**不需要安裝任何軟體！**所有工具都在雲端。

### 如果您想在自己電腦上開發

#### 1. Node.js（必要）
- **用途**：執行 JavaScript 程式
- **下載**：https://nodejs.org/
- **版本**：建議下載 LTS 版本（長期支援版）

#### 2. Git（必要）
- **用途**：版本控制
- **下載**：https://git-scm.com/
- **測試是否安裝成功**：打開終端機，輸入 `git --version`

#### 3. Visual Studio Code（推薦）
- **用途**：程式碼編輯器
- **下載**：https://code.visualstudio.com/
- **建議安裝的擴充套件**：
  - Firebase
  - ESLint
  - Prettier

#### 4. Firebase CLI（必要）
安裝 Node.js 後，在終端機執行：
```bash
npm install -g firebase-tools
```

測試是否安裝成功：
```bash
firebase --version
```

## 📱 手機（測試用）

**為什麼需要手機？**
- 測試 LIFF（必須在手機 LINE 上測試）
- 測試 GPS 簽到功能
- 測試 LINE Login 行動版

**建議準備：**
- 至少一支安裝 LINE 的手機（iPhone 或 Android 都可以）
- 最好準備兩支（一支測試一般使用者，一支測試管理員）

## 🗂️ 資料整理工具

### 建議準備一個筆記本

**記錄什麼？**
1. **帳號資訊**
   - LINE Channel ID
   - Firebase 專案 ID
   - GitHub 倉庫網址
   
2. **密鑰（加密保存！）**
   - LINE Channel Secret
   - Firebase Service Account Key
   - 各種 API Keys
   
3. **網址**
   - 正式環境網址
   - 測試環境網址
   - Firebase Console 連結
   
4. **開發紀錄**
   - 遇到的問題
   - 解決方法
   - 重要的設定值

**推薦工具：**
- **實體筆記本**：最安全，但可能會弄丟
- **Notion**：方便搜尋和整理
- **Google Docs**：隨時隨地可以查看
- **1Password / LastPass**：專門管理密碼和密鑰（最推薦！）
